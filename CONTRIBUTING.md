# Contributing

Project Aurora is pre-alpha. Before opening a pull request, read the README,
architecture, safety guidance, and `AGENTS.md`. Keep changes narrowly scoped,
avoid secrets and machine-specific configuration, and preserve configurable
hardware assumptions.

Run `make check` before submitting. Include tests for functional changes and
update documentation when behavior or architecture changes. Hardware proposals
must not invent electrical ratings, wiring instructions, or safety claims.
